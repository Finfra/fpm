# 실행 절차

인자: `$ARGUMENTS` → `<프로젝트번호> [mode]`

## 0. 인자 검증

* 인자 없음 → Usage + `cdf` 현황 출력 후 종료
* 필수 1개 파라미터 (`프로젝트번호`) + 선택 1개 파라미터 (`mode`)
* `mode` 기본값: `backup`
* `mode` 허용값:
    - `backup` — 폴더를 `~/_git/z_backup/`로 mv + 레지스트리 정리
    - `done` — 완료된 프로젝트를 `~/_git/z_done/`로 mv + 레지스트리 정리
    - `keep` — 레지스트리만 정리, 폴더 유지
* 그 외 mode → 에러 후 Usage 출력

## 1. 사전 확인

| #   | 항목                 | 동작                                                                     |
| :-- | :------------------- | :----------------------------------------------------------------------- |
| 1-1 | 번호 유효성 검증     | `projects/{번호}` 파일 존재 여부 확인. 없으면 에러 후 종료               |
| 1-2 | 시스템 프로젝트 검사 | 번호 `0`, `1`, `2`, `3`은 **삭제 불가** — 에러 후 종료                   |
| 1-3 | 프로젝트 정보 표시   | 번호, 프로젝트명, 경로, 타입을 `Projects.md`에서 조회하여 표시           |
| 1-4 | 사용자 컨펌          | 삭제 대상 정보를 보여주고 **반드시 사용자 승인** 후 진행                  |

## 2. 제거 실행

| #   | 항목                        | 동작                                                                                  |
| :-- | :-------------------------- | :------------------------------------------------------------------------------------ |
| 2-1 | 프로젝트 폴더 이동          | **mode=backup**: 원본 → `~/_git/z_backup/{프로젝트명}_{YYYYMMDD}` 로 `mv`. **mode=done**: 원본 → `~/_git/z_done/{프로젝트명}_{YYYYMMDD}` 로 `mv`. **mode=keep**: 폴더 유지 |
| 2-2 | `projects/{번호}` 파일 삭제 | `___pm/projects/{번호}` 파일 제거                                                     |
| 2-3 | `Projects.md` 업데이트      | 테이블에서 해당 행 제거 + setting Script에서 해당 echo 라인 제거                      |
| 2-4 | `Harness.md` 정리           | `___pm/Harness.md`에서 해당 프로젝트 엔트리 제거                                      |
| 2-5 | `pm_history/` 기록          | `_doc_work/pm_history/{YYYY-MM-DD}-{id}-del-{번호}-{프로젝트명}.md` 생성 (frontmatter `mode` 포함) |
| 2-6 | 결과 리포트                 | 모드, 이동된 경로(있으면), 제거된 엔트리 목록 출력                                    |

## 이동 경로 정책

* **backup 경로**: `~/_git/z_backup/{프로젝트명}_{YYYYMMDD}`
* **done 경로**: `~/_git/z_done/{프로젝트명}_{YYYYMMDD}` — 완료된(종결된) 프로젝트 보관
* 동일 이름 존재 시 `_{YYYYMMDD}_{N}` 서픽스 추가 (ex: `myApp_20260411_2`)
* **절대 삭제 금지**: `rm -rf` 등 원본 폴더 삭제 명령 사용 불가
* 대상 폴더(`z_backup/` 또는 `z_done/`)가 없으면 자동 생성

## 안전장치

* 번호 `0` (홈), `1` (___pm), `2` (obsidian), `3` (claude)은 **시스템 프로젝트**로 삭제 불가
* 사용자 컨펌 없이 절대 진행하지 않음
* `backup`/`done` 모드에서 mv 실패 시 후속 단계(2-2~2-5) 진행하지 않음
* `keep` 모드는 폴더를 건드리지 않음

## Usage (인자 없을 때)

```
Usage: /pm-del <프로젝트번호> [mode]

mode:
  backup  (기본) 폴더를 ~/_git/z_backup/ 으로 mv + 레지스트리 정리
  done    완료된 프로젝트 폴더를 ~/_git/z_done/ 으로 mv + 레지스트리 정리
  keep    폴더는 그대로 두고 ___pm 레지스트리만 정리

예시:
  /pm-del 42
  /pm-del 15 backup
  /pm-del 15 done
  /pm-del 15 keep
```

이후 `cdf` (인자 없이) 실행 결과를 표시하여 등록된 프로젝트 목록 확인 가능하게 함.
