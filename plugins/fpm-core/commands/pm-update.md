# 실행 절차

인자: `$ARGUMENTS` → `<프로젝트번호> [대상]`

## 0. 인자 검증

* 인자 없음 → Usage + `cdf` 현황 출력 후 종료
* 필수 1개 파라미터 (`프로젝트번호`) 확인
* 선택 1개 파라미터 (`대상`): `all` / `scar` / `template` / `folder` (기본: `all`)

## 1. 사전 확인

| #   | 항목               | 동작                                                          |
| :-- | :----------------- | :------------------------------------------------------------ |
| 1-1 | 번호 유효성 검증   | `projects/{번호}` 파일 존재 여부 확인                         |
| 1-2 | 프로젝트 정보 조회 | `Projects.md`에서 타입(Domain), 경로, 프로젝트명 조회         |
| 1-3 | 현재 상태 스캔     | 대상 프로젝트의 파일/폴더/SCAR 현황을 수집                    |

## 2. diff 생성 및 컨펌

| #   | 항목             | 동작                                                                   |
| :-- | :--------------- | :--------------------------------------------------------------------- |
| 2-1 | 변경 사항 계산   | pm-new 기준 vs 현재 상태를 비교하여 추가/변경/누락 항목 리스트 생성    |
| 2-2 | diff 리포트 출력 | 항목별로 `[추가]`, `[변경]`, `[유지]` 표시                             |
| 2-3 | 사용자 컨펌      | **변경이 있는 모든 항목**에 대해 일괄 또는 개별 승인 요청               |

> **핵심 원칙**: 기존 파일이 존재하면 무조건 사용자 컨펌. 자동 덮어쓰기 절대 금지.

## 3. 업데이트 실행

### template 업데이트 (대상: `template` 또는 `all`)

| #   | 항목                    | 동작                                                                   |
| :-- | :---------------------- | :--------------------------------------------------------------------- |
| 3-1 | `.gitignore`            | 템플릿과 비교하여 누락 항목만 추가 (기존 항목 유지)                    |
| 3-2 | `Issue.md`              | 섹션 구조만 비교. **기존 이슈 데이터는 절대 수정 불가**                |
| 3-3 | `noteForHuman.md`       | 없으면 생성. 있으면 컨펌 후 업데이트                                   |
| 3-4 | `PROMPTS.md`            | 없으면 생성. 있으면 컨펌 후 업데이트                                   |
| 3-5 | `Harness.md`            | global layer를 최신 SCAR 목록과 비교하여 누락분 추가. 기존 내용 유지   |
| 3-6 | `.vscode/settings.json` | peacock.color 등 기존 설정 유지. 누락 키만 추가                        |
| 3-7 | `CLAUDE.md`             | 있으면 **스킵** (프로젝트 고유 문서이므로 자동 업데이트 안 함)         |

### folder 업데이트 (대상: `folder` 또는 `all`)

| #   | 항목             | 동작                                                    |
| :-- | :--------------- | :------------------------------------------------------ |
| 3-8 | 공통 폴더        | `_doc_work/{tasks,report,plan,history,z_done}`, `_doc_arch/z_old`, `.claude/` — 없는 것만 생성 |
| 3-9 | 타입별 추가 폴더 | pm-new 기준 타입별 폴더 목록과 비교하여 없는 것만 생성 (mac: `_doc_work/_release/`, `_doc_arch/define/`) |

### scar 업데이트 (대상: `scar` 또는 `all`)

| #    | 항목                   | 동작                                                               |
| :--- | :--------------------- | :----------------------------------------------------------------- |
| 3-10 | 프로젝트 `Harness.md` | global SCAR 목록을 최신화 (동일 타입 프로젝트에서 수집)            |
| 3-11 | `___pm/Harness.md`    | 해당 프로젝트 엔트리가 없으면 추가. 있으면 diff 후 컨펌            |
| 3-12 | `Projects.md` 검증    | 누락 컬럼이나 불일치 있으면 보고 (자동 수정은 컨펌 후)             |

## 4. 결과 리포트

| #   | 항목                 | 동작                                                           |
| :-- | :------------------- | :------------------------------------------------------------- |
| 4-1 | 변경 요약 출력       | `[추가됨]`, `[업데이트됨]`, `[스킵됨]` 항목별 리스트           |
| 4-2 | `pm-history.md` 기록 | `_doc_work/pm-history.md`에 실행 이력 append                   |
| 4-3 | 커밋 제안            | 변경 사항이 있으면 커밋 여부를 사용자에게 문의                  |

## 컨펌 정책

| 상황                    | 동작                                    |
| :---------------------- | :-------------------------------------- |
| 파일이 없음 (신규 생성) | 자동 생성 (컨펌 불필요)                 |
| 파일이 존재 + 변경 없음 | `[유지]` 표시, 스킵                     |
| 파일이 존재 + 변경 있음 | diff 표시 후 **반드시 사용자 컨펌**     |
| 폴더가 없음 (신규 생성) | 자동 생성 (컨펌 불필요)                 |
| 폴더가 존재             | 스킵                                    |
| SCAR 엔트리 추가        | 추가 내용 표시 후 컨펌                  |
| SCAR 엔트리 변경        | 변경 전후 표시 후 **반드시 사용자 컨펌** |

## Usage (인자 없을 때)

```
Usage: /pm-update <프로젝트번호> [대상]

대상:
  all       전체 업데이트 (기본값)
  scar      SCAR 엔트리만 최신화
  template  템플릿 기반 파일만 업데이트
  folder    타입별 폴더 구조만 보완

예시:
  /pm-update 15
  /pm-update 15 scar
  /pm-update 15 template
```

이후 `cdf` (인자 없이) 실행 결과를 표시하여 등록된 프로젝트 목록 확인 가능하게 함.
