# Project Overview

A Project Management repository. It manages the paths of projects under `~/_git/__all/` via a numeric index, and lets you jump quickly between directories through the zsh function `cdf`.

# Structure

* `projects/` - numbered files (0~57); each file stores the target directory path on a single line (full list: `Projects.md`)
    - `0` → `~` (home directory)
    - `1` → `~/_git/___pm` (this repository itself)
    - `2` → `~/_doc` (Obsidian vault)
    - `3` → `~/.claude`
    - `11`~`16` → managed macOS App (fApp) project paths
    - `51`~`57` → CLI / libraries
    - **Auto-generated** — regenerated by running `fpm-projects-sync` after editing `Projects.md` (the SSOT); do not edit directly.
* `sh/` - shell tools (public-mirror target, the single SSOT install payload — Issue149). The old `shell/fpm-functions.zsh` duplication has been removed. Only the per-machine artifact `fpm_aliases_iterm-bg.sh` is gitignored. **Install-location agnostic (`$FPM_BASE`-based) — works wherever you put it, e.g. `~/_git/___pm` or `~/_git/__all/fpm`.**
    - `fpm.sh` - **bootstrap**: determines `$FPM_BASE` (env first, otherwise self-detects its own location), then loads `fpm_function.sh` + `fpm_aliases.sh`. In `~/.zshrc` / `~/.bashrc`, `export FPM_BASE=...` then source it.
    - `fpm_function.sh` - cdf/cdff/cdfc/cdfv/cdft + sshf navigation functions (`$FPM_BASE`-based)
    - `fpm_aliases.sh` - collection of aliases (sources the generated iterm-bg alias file)
    - `fpm_aliases_iterm-bg.sh` - **auto-generated** (gitignored). Produced by `update-iterm-bg`, loaded by `fpm_aliases.sh`.
    - `fpm-projects-sync` - **unified driver**: applies `Projects.md` → `projects/` index + each `.vscode` background color (peacock) / emoji + iterm-bg aliases in one batch (a single manual command, `$FPM_BASE`-based)
    - `update-iterm-bg` - regenerates `Projects.md` color → `$FPM_BASE/sh/fpm_aliases_iterm-bg.sh` aliases (driver step [3/3]. Previously it only generated `~/.zsh_aliases_iterm-bg.sh` without loading it → moved inside the install folder so `fpm_aliases.sh` sources it).
* `README.md` - **fpm public-mirror only** product introduction document. It is not kept in the ___pm body (to avoid conflicts) — `publishable-policy.yml` `exclude[]` protects the fpm copy from forward `--delete` and also blocks reverse absorption. Edit the README in fpm (prj7).

# cdf Function Family (core mechanism)

Shell functions defined in `sh/fpm_function.sh` (sourced by the bootstrap `sh/fpm.sh`). The shared helper `_pm_manager()` uses `$FPM_BASE/projects` as its base path (falls back to the legacy `~/.info/__pmBasePath.txt` when FPM_BASE is unset).

| Function | Description                                                       | Example                   |
| :------- | ---------------------------------------------------------------- | ------------------------- |
| `cdf`    | Move terminal directory (splits iTerm2 horizontally for multiple indexes) | `cdf 10` / `cdf 11 12 13` |
| `cdff`   | Open the path in Finder                                          | `cdff 14`                 |
| `cdfc`   | Copy the path to the clipboard                                   | `cdfc 2`                  |
| `cdfv`   | Open the path in VS Code (multiple allowed)                      | `cdfv 0 1 2 10`           |

* Run with no arguments to print the full project list.
* Returns an error if the index file does not exist.

# Adding / Changing Projects

Create a numbered file under the `projects/` folder and record the path on a single line. Also update `Projects.md` (the SSOT).
